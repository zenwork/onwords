# On Words Development Guide

## Project

- This is a Deno Fresh application.
- Use `deno task start` to run the development server. Do not use `deno run start`; it is not a configured command.
- Fresh treats every file under `routes/` as a route. Put shared server utilities in `lib/` or another non-route directory.
- `fresh.gen.ts` is generated by Fresh. Do not hand-maintain its route list; start the development server to regenerate it after route changes.

## Change Workflow

1. Before editing, inspect `git status --short` and preserve unrelated working-tree changes.
2. Keep changes scoped to the requested behavior. Update imports and generated manifest output when files are moved into or out of `routes/`.
3. Run the verification commands below after every code change:

```sh
deno lint
deno check main.ts
git diff --check
```

4. Run `deno fmt --check <changed-file>` for each changed Deno or configuration file that was formatted as part of the task. Do not reformat unrelated legacy files merely to satisfy a repository-wide check.
5. For every change that can affect the running site, run a startup smoke test: start `deno task start`, wait for Fresh to reach its listening state, then request the Local URL reported by Fresh. The request must return HTTP 200 before the work is considered complete. Stop the watcher after the check.
6. Report the commands run and any validation that could not run because of the environment.
